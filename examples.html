The DOM

The DOM specifies how the browser should create a model of an HTML page and how JavaScript can access and update the content of a web page while it is in the browser window

The DOM is separate set of rule meaning it is neither HTML nor JavaScript

Making a model of the HTML page

When a browser opens a page it creates a model of the page in the memory.

The DOM tree

As a browser loads a page , it creates a model of that page. The model is called a DOM tree, and it is stored in the browser' memory

It consists of four main types of nodes.

  • The document node

    Every element, attribute, and piece of text in HTML is represented by its own DOM node in the tree. At the top of the tree is always a document node, it represents the entire page and it is becouse of this when you access any element, attribute or text you navigate to them via the document node. It is the starting point of all visits to the DOM tree.

  • Elements Nodes

    HTML elements describe the structure of an HTML page
    e.g.
    h1...h6 describe headings
    p describes paragraphs.
    We access the the DOM tree using elements and once you find the element you want the you can access its text and attribute nodes if you want.

  • Attribute node

    Attributes are included in the opening tag of HTML elements and they are represented as attribute nodes in the DOM tree. The become part of the element in which they are included but not chidrens. There are some specific JavaScript methods and properties used to read or change elements attributes

  • Text nodes

    Ones you have accessed an element node, you can then reach the text within that element. This is stored in its own text node.

Working with the DOM tree

Accessing the DOM tree involves two steps:

  1. Locate the node that represents the element you want to work with.
  2. Use its text content,child element, and attributes.

Access the elements

In this section we look at some of the methods and properties used o access the elements

  1. Methods and properties that select an individual element node

    • getElementById()
    • querySelector()

  2. Select multiple elements (NODELISTS)

    • getElementByClassName()
    • getElementByTagName()
    • getSelectorAll()

  3. Traversing between elements nodes

    • parentNode
    • priviosSibling / nextSibling
    • firstChild / lastChild

Work with those elements

  1. Access or update text node

    • nodeValue

  2. Work with HTML content

    • innerHTML
    • textContent
    • createElement()
    • createTextNode()
    • appendChild() / removeChild()

  3. Access or update attribute values

    • className / id
    • hasAttribute()
    • getAttribute()
    • setAttribute()
    • removeAttribute()

      • Treversing the Dom can be difficult because some browsers add a text node whenever they come across whitespace between elements. A good example is internet explore, so most of these properties would not work as expected in this browsers.

      One of the most popular way to address this kind of problem is to use a JavaScript library such as jQuery, which helps dealwith such problems.

How to get / update element content

So far we have forcused on finding elements in the DOM tree. Now we focus on how to access / update element content

To work with the content of elemnts you can:

  • Navigate to the text nodes. This works best when the element contains only text, no other elements.

  • Work with the containing element. This allows you to access itstext nodes and child elements. It works better when an element has text nodes and elements that arre siblings.

Text nodes

Ones you have navigated from an element to its textnode, there is one property that you will commonly find yourself using:
nodeValue //Used to access text from node

Containing element

When you are working with an element node, that element can contain markup. You have to choose whether you want to retriev (get) or update (set) the markup as well as the text.

innerHTML //Gets/sets text & markup
textContent //Gets/sets text only
innerText //Gets/sets text only
